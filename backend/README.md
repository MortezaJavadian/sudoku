# بک‌اند حل‌کننده سودوکو (Sudoku Solver Backend)

این پروژه شامل بک‌اند پایتون برای حل‌کننده‌ی سودوکو است. وظیفه‌ی این بخش پردازش تصویر، تشخیص ارقام و حل معمای سودوکو است.

## راه‌اندازی (Setup)

۱. مطمئن شوید که Python 3 نصب شده است.
۲. محیط مجازی (Virtual Environment) را بسازید و فعال کنید:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
۳. پیش‌نیازها را نصب کنید:
   ```bash
   pip install -r requirements.txt
   ```

## ساختار پروژه
پروژه به شکل زیر سازماندهی شده است تا برای فازهای بعدی و اضافه شدن دیتاست‌های جدید کاملاً آماده باشد:
- `data/`: شامل دیتاست‌ها و خروجی‌ها.
  - `sudoku_images/`: تصاویر نمونه (از آسان تا سخت) برای تست گرفتن از الگوریتم.
  - `mnist/`: پوشه‌ای برای قرارگیری دیتاست ارقام در فاز بعدی.
  - `output/`: خروجیِ اسکریپت‌های تست، دسته‌بندی شده با نام دیتاست مربوطه (مانند `output/sudoku_images/`).
- `src/`: سورس‌کدهای اصلی پروژه (مثل `extractor.py`).
- `tests/`: اسکریپت‌های تست برای بررسی عملکرد فازهای مختلف روی کل دیتاست‌ها (مثل `test_phase1.py`).

## فاز ۱: استخراج جدول (Grid Extraction)

در این فاز هدف پیدا کردن دقیق جدول سودوکو در یک عکس، از بین بردن زاویه (Perspective Transform) برای تبدیل آن به یک مربع کاملاً صاف، و نهایتاً برش آن به ۸۱ خانه‌ی مجزا است.

### مراحل پیاده‌سازی فاز ۱
1. **تبدیل به خاکستری و نویزگیری:** تصویر با فیلتر گوسی مات شد تا نویزها کاهش یابند.
2. **استخراج لبه‌ها (Adaptive Threshold):** تصویر به حالت باینری درآمد تا خطوط روشن شوند.
3. **یافتن سلول‌های شبکه‌ای:** با استفاده از درختِ سلسله‌مراتب کانتورها (Hierarchy)، ۸۱ سلول مجزا پیدا شدند تا نویزها و نوشته‌های اضافی کاملا فیلتر شوند.
4. **تراز کردن محورها (Axis-Aligned Rotation):** کل شبکه‌ی سلول‌ها به طور هندسی چرخانده شد تا کاملا صاف شود.
5. **برازش خطی لبه‌ها (Linear Edge Fitting):** چهار خطِ مستقیم برای چهار لبه‌ی (بالا، پایین، چپ، راست) جدول به صورت ریاضی تخمین زده شد.
6. **تلاقی ریاضی:** محل تقاطع دقیق این ۴ خط به عنوان چهار گوشه‌ی اصلی جدول محاسبه شد.
7. **Warp Perspective:** چهارگوشه جدول به یک مربع صاف تبدیل شد.
8. **برش خانه‌ها (Cell Cropping):** مربع به ۸۱ خانه مساوی تقسیم شد.

### مشکلاتی که در طول مسیر با آن‌ها برخوردیم و راه‌حل‌ها:
- **مشکل Padding:** در ابتدا یک حاشیه سفید برای تصویر در نظر گرفتیم که در عکس‌های شلوغ باعث تشخیص اشتباه کادر می‌شد. با حذف آن و اتکا به `RETR_TREE` مشکل حل شد.
- **مشکل برش خطوط ضخیم جدول:** برش با مقدار ثابت (مثلاً ۴ پیکسل) باعث می‌شد در رزولوشن‌های مختلف حاشیه‌های مشکی وارد سلول شوند. برای حل این مشکل، میزان بُرش به صورت **پویا (۱۰٪ از عرض و طول سلول)** محاسبه شد.
- **چالش عکس‌های ۰۷، ۰۸ و ۰۹ (زاویه شدید، حاشیه چسبیده، پارگی گوشه‌ها):** در عکس‌هایی مثل `07.jpg` که یک حاشیه قرمز به جدول چسبیده بود و یا گوشه‌ی پایین-چپ عکس پاره شده و وجود نداشت، پیدا کردن کانتورِ بیرونی باعث جمع‌شدگی و به هم ریختگی شدیدِ اعوجاج تصویر (Distortion) می‌شد.
  - *راه‌حل نهایی:* الگوریتم **برازش خطی لبه‌ها (Edge Fitting)** پیاده‌سازی شد. به این شکل که ابتدا سلول‌های سالم شناسایی می‌شوند و سپس خطوط لبه‌ی جدول از روی چیدمانِ این سلول‌ها عبور داده می‌شوند. در نتیجه حتی اگر یک گوشه‌ی بزرگ از جدول کنده شده باشد، تقاطعِ خطوط ریاضی آن را دقیقاً بازسازی می‌کند و خطای پرسپکتیو را به ۰ می‌رساند.

### اجرای تست‌های فاز ۱
برای اجرای تست‌های فاز اول روی تمامی عکس‌های سودوکو و تولید مجدد خروجی‌ها (از جمله `debug_grid.jpg` و `debug_thresh.jpg`)، دستورات زیر را اجرا کنید:

```bash
cd tests
python test_phase1.py
```
با این کار تمامی عکس‌های مسیر `data/sudoku_images/` خوانده شده و نتایج ۱۰۰٪ دقیق آن‌ها در مسیر `data/output/sudoku_images/` پوشه‌بندی می‌شوند.

## فاز ۲: تشخیص ارقام (Digit Recognition)

در این فاز هدف طراحی و آموزش یک مدل شبکه‌ی عصبی (CNN) برای تشخیص ارقام داخل سلول‌ها است. مدل باید بتواند اعداد ۱ تا ۹ را به همراه کلاس ۰ (خانه‌ی خالی) تشخیص دهد.

### مراحل پیاده‌سازی فاز ۲
1. **آماده‌سازی دیتاست:** دانلود دیتاست MNIST برای ارقام ۱ تا ۹ و تولید کلاس صفر (برای خانه‌های خالی) از روی عکس‌های واقعی سودوکو.
2. **طراحی شبکه عصبی (CNN):** طراحی یک شبکه‌ی سبک و سریع با فریم‌ورک PyTorch.
3. **آموزش مدل (Training):** استفاده از Data Augmentation برای مقاوم‌سازی مدل در برابر فونت‌های مختلف و نویز.
4. **ارزیابی (Evaluation):** بررسی مدل با استفاده از Confusion Matrix و نمودارهای Loss/Accuracy.

### مشکلات برطرف شده و نتایج تست فاز ۲:
- **دقت ۹۹ درصدی ترکیبی:** مدل توانست بر روی دیتاست ادغام‌شده‌ی اعداد فارسی (Hoda)، انگلیسی (MNIST) و سلول‌های خالی، روی بیش از ۲۸,۰۰۰ تصویر تست به دقت چشم‌گیر **۹۸.۹۶٪** دست یابد.
- **حل چالش عدد ۰:** از آنجا که سودوکو عدد ۰ ندارد، کلاس ۰ کاملاً به عنوان خانه‌ی خالی به شبکه معرفی شد و در نتیجه مدل ۱۰۰٪ مواقع خانه‌های خالی را به‌درستی تشخیص داد (بدون اشتباه گرفتن با نویز یا اعداد).
- شبکه به عنوان فایل `backend/models/digit_recognizer.pth` ذخیره شده است.
